COMPREHENSIVE DEVELOPMENT REPORT

The Hill Cipher Implementation

This report details the design, implementation, and verification of the **Screener application**, a web-based tool for the Hill Cipher. The project successfully integrated **Python Flask** and **NumPy** to provide a robust, matrix-based cryptographic utility.

Project Team: Rethish, Praveen, Vishal, Samiksha, Yuvasree, Selvapriya, Pavatharani

Modulus: 128 (Full ASCII Support)

PHASE 1: Mathematical Foundations

The Hill Cipher operates on linear algebra, transforming plaintext into ciphertext using a **Key matrix (K)**. A critical focus was enforcing strict **Modulo 128 arithmetic** to ensure decryption accuracy across the full ASCII character set.

Character Encoding & Modulus

The application uses the full range of ASCII characters. Plaintext characters are mapped to integers in the range **[0, 127]** (Encoding). A modulus (m) of 128 is applied to all results, ensuring the ciphertext is always valid ASCII (Modulus).

M $\in$ [0, 127] (mod 128)

Key Invertibility Condition

The success of decryption hinges on the existence of $K^{-1}$ (mod 128). The key matrix must be invertible for decryption to succeed.

  • **Condition:** This requires that $\text{GCD}(det(K), 128)$ must equal 1. Since $128 = 2^7$, the determinant must be an odd integer.
  • **Implementation:** The modular inverse is calculated using the Extended Euclidean Algorithm on the determinant, followed by the classical adjoint matrix method. This calculation is performed entirely within the server logic.
Example Invertible Key (2x2): $K = \begin{pmatrix} 3 & 5 \\ 2 & 3 \end{pmatrix}$

**Padding:** Automatic padding ensures the message length is a multiple of the key dimension (N), essential for matrix multiplication. Padding is automatically stripped during decryption.

PHASE 2: Encryption - The Matrix Transformation

Plaintext vectors (**P**) are transformed by multiplication with the Key matrix (**K**). The NumPy library is utilized for highly efficient linear algebra operations.

The Encryption Formula

Plaintext vectors (P) are transformed by the Key matrix (K) to produce the Ciphertext (C).

C = (P K) (mod 128)

PHASE 3: Decryption - Reversing the Cipher

Ciphertext (**C**) is reversed back into the original Plaintext (**P**) using the **modular inverse of the key** ($K^{-1}$). The inverse is calculated using the Extended Euclidean Algorithm on the determinant.

The Decryption Formula

The original plaintext P is recovered by multiplying the ciphertext C by the modular inverse of the Key $K^{-1}$, modulo 128.

P = (C K-1) (mod 128)

**Secure Output Format:** To prevent trivial substitution, the ciphertext is presented as a numerical array (e.g., [69, 58, 47, ...]) rather than a direct string of characters, which is required as input for the decryption function.

Future Vectors: Enhancements

The application's future scope includes crucial enhancements to further improve its utility and demonstration capabilities:

  • **Automated Key Generation:** Implementation of a utility that guarantees modular invertibility for key creation.
  • **Frequency Analysis Tool:** Addition of a tool to illustrate cryptanalysis techniques against the cipher.
  • **Custom Alphabet Mapping:** Expansion to support custom character mapping beyond the standard ASCII range.
